# Web/アプリ開発 法的コンプライアンスチェック

Webサイトやアプリケーションを開発・公開する際に確認すべき法的リスクと対策をまとめる。特に「特定の商品や企業を紹介するサービス」において重要となる。

## 1. 著作権と画像利用
他社の商品画像や説明文を利用する場合のルール。

### 1-1. 原則（NG行為）
*   **無断転載**: ネット上の画像や公式サイトのスクリーンショットを、許可なく自分のアプリやサイトに掲載することは原則として著作権侵害となる。
*   **「引用」の範囲外**: 商品画像をメインビジュアルとして大きく使い、自分のコメントが少ない場合は「引用」とは認められにくい（主従関係の逆転）。

### 1-2. クリアするための対策
1.  **許諾を取る（推奨）**:
    *   対象企業（農家さん等）に企画を説明し、「公認アプリ」として画像利用の許可をもらう。これが最も安全で、ビジネス的にもプラスになる。
2.  **アフィリエイトASPを利用する**:
    *   A8.net等のASPを通じて提携し、提供された「商品リンク（画像付き）」を使用する。これは許諾済みの利用となる。
3.  **引用の要件を厳守する（非営利・ファンサイトの場合）**:
    *   自分の文章（レビューや紹介）を「主」とし、画像を「補足」として小さく扱う。
    *   出典元（公式サイト名とURL）を画像の直下に明記する。
4.  **自分で撮影・生成する**:
    *   自分で商品を購入して撮影した写真は、自分の著作物として自由に使える（ただし写り込みや商標には注意）。
    *   AI生成画像で「イメージ」として表現し、特定商品の写真を使わない（今回の実装パターン）。

## 2. 必須記載事項（サイト下部等）

### 2-1. プライバシーポリシー
ユーザー情報を取得する場合は必須。取得しない場合でも「取得していない」旨や、Google Analytics等の利用有無を記載すべき。
*   **項目例**: 収集する情報、利用目的、第三者提供の有無、問い合わせ先。

### 2-2. 免責事項
トラブル回避のために必須。
*   **内容例**:
    *   「本アプリの診断結果は医学的なアドバイスではありません」
    *   「紹介している商品は、リンク先の企業と取引を行ってください」
    *   「本アプリの利用による損害について、運営者は責任を負いません」

## 3. 営利目的の場合の法律

### 3-1. 特定商取引法に基づく表記
*   自分で商品を販売する場合、または「ドロップシッピング」等の場合は必須。
*   住所、電話番号、代表者名などの公開義務がある。
*   ※アフィリエイト（紹介のみ）の場合は、自分は販売者ではないため不要なケースが多いが、誤認されないような表記が必要。

### 3-2. 景品表示法（ステマ規制）
*   広告収入（アフィリエイト報酬等）を得て掲載する場合、**「PR」「広告」「プロモーション」**等の表記を分かりやすく入れる義務がある（2023年10月施行）。

## 4. このアプリ（AI梅コンシェルジュ）の場合の推奨アクション
1.  **AI画像の使用**: 現在のプロトタイプのように、特定の「商品写真」ではなく「イメージイラスト」を使うのは著作権リスクが低い良いアプローチ。
2.  **「非公式」の明記**: 濱田農園様の公認でない場合は、「本アプリは濱田農園様のファンによる非公式ファンアプリです」と明記する。
3.  **リンク誘導**: 購入は公式サイトへ誘導しているため、販売責任のリスクは低い。ただし、リンク先が変わる可能性があるので定期チェックが必要。
